## Проблема

Нефтегазовые компании ежегодно теряют миллиарды тенге из-за производственных инцидентов. Данные о происшествиях и нарушениях накапливаются, но не анализируются системно — менеджеры по безопасности работают реактивно, а не проактивно.

## Решение

HSE AI Dashboard превращает накопленные данные об инцидентах в **предсказуемые паттерны и конкретные действия**. Система предсказывает риски за месяцы до инцидентов, автоматически расставляет приоритеты и генерирует рекомендации через Claude AI.

---

## Что умеет платформа

### 1. Обзор — сводная панель
Единый экран с ключевыми KPI: количество инцидентов, прогноз на год, потенциальная экономия, активные алерты по организациям.

### 2. Анализ происшествий
- Статистика по типам, организациям, локациям и причинам
- **Прогноз инцидентов на 3/6/12 месяцев** с 80% доверительным интервалом
- Топ-5 зон риска с индексом опасности

### 3. Карта Коргау — поведенческие аудиты
- Статистика наблюдений: нарушения vs хорошие практики
- **Рейтинг организаций** (индекс риска 0–100)
- Корреляция нарушений с последующими инцидентами
- 4-уровневые алерты: Критический / Высокий / Средний / Низкий

### 4. AI Рекомендации
5 конкретных мер безопасности, сгенерированных Claude на основе актуальных данных системы. Каждая — с приоритетом, целевой аудиторией и ожидаемым эффектом.

### 5. Экономический эффект
Расчёт ROI внедрения: предотвращённые инциденты, сэкономленные тенге и USD, срок окупаемости в месяцах.

### 6. AI Аналитик — чат с данными ⭐
Задайте вопрос на русском языке и получите ответ на основе **реальных данных** вашей системы:

```
"Какие организации имеют критический уровень риска?"
"Почему выросли нарушения по СИЗ в последние месяцы?"
"Что нужно сделать org_03 чтобы снизить аварийность?"
"Есть ли корреляция между нарушениями LOTO и авариями оборудования?"
```

## Локальный запуск

### Backend
```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # добавьте ANTHROPIC_API_KEY
python data/generate.py       # генерация синтетических данных
uvicorn main:app --reload
# → http://localhost:8000/docs
```

### Frontend
```bash
cd frontend
npm install
# .env.local уже содержит NEXT_PUBLIC_API_URL=http://localhost:8000
npm run dev
# → http://localhost:3000
```

### Переменные окружения

**Backend `.env`:**
```
ANTHROPIC_API_KEY=sk-ant-...   # обязателен для /recommendations и /chat
```

**Frontend `.env.local`:**
```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

## Данные

Система работает на **синтетических данных**, сгенерированных специально для демо:

- **600 инцидентов** за 2023–2025 годы по 7 организациям и 13 типам локаций
- **1200 карточек Коргау** (поведенческие аудиты: нарушения и хорошие практики)
- Данные воспроизводимы (`random.seed(42)`) и регенерируются при каждом старте бэкенда

В продакшене CSV-слой заменяется подключением к СУБД компании.

## Алгоритмы

**Прогнозирование инцидентов**
Линейный тренд + месячная сезонность (коэффициенты по историческим данным). Доверительный интервал 80% (±1.28σ). Горизонт: 3/6/12 месяцев.

**Risk Scoring**
Взвешенный индекс риска на организацию: 60% — тяжесть инцидентов (НС=10, Авария=7, ...), 40% — количество нарушений по Коргау. Нормализация 0–100.

**Алерты (4 уровня)**
- 🔴 Критический: нарушений >2× среднемесячного за последние 30 дней
- 🟠 Высокий: один тип нарушения повторился >3 раз за 30 дней
- 🟡 Средний: рост нарушений >15% к аналогичному периоду прошлого года
- 🟢 Низкий: снижение нарушений >20% (положительный сигнал)

**AI Рекомендации и Чат**
Claude получает сводку последних 6 месяцев данных (топ типов инцидентов, причин, зон риска) и генерирует конкретные меры. Кэш рекомендаций — 1 час. Чат поддерживает историю диалога.
