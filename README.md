## Проблема

Нефтегазовые компании ежегодно теряют миллиарды тенге из-за производственных инцидентов. Данные о происшествиях и нарушениях накапливаются, но не анализируются системно — менеджеры по безопасности работают реактивно, а не проактивно.

## Решение

HSE AI Dashboard превращает накопленные данные об инцидентах в **предсказуемые паттерны и конкретные действия**. Система предсказывает риски за месяцы до инцидентов, автоматически расставляет приоритеты и генерирует рекомендации через Claude AI.

## Что умеет платформа

### 1. Обзор — сводная панель
Единый экран с ключевыми KPI: количество инцидентов, прогноз на год, потенциальная экономия (~197 млн тенге), активные алерты по организациям.

### 2. Анализ происшествий
- Статистика по типам, организациям, локациям и причинам
- **Прогноз инцидентов на 3/6/12 месяцев** с 80% доверительным интервалом (Holt-Winters ETS, R²=0.796)
- Топ-5 зон риска с взвешенным индексом опасности

### 3. Карта Коргау — поведенческие аудиты
- Статистика наблюдений: нарушения vs хорошие практики
- **Рейтинг организаций** (индекс риска 0–100)
- Корреляция нарушений с последующими инцидентами (r=0.407)
- 4-уровневые алерты: Критический / Высокий / Средний / Низкий

### 4. AI Рекомендации
5 конкретных мер безопасности, сгенерированных Claude на основе актуальных данных системы. Каждая — с приоритетом, целевой аудиторией и ожидаемым эффектом. Кэш 1 час.

### 5. Экономический эффект
Расчёт ROI: предотвращённые инциденты (~103/год), сэкономленные тенге (~197 млн/год, ~411K USD), срок окупаемости.

### 6. AI Аналитик — чат с данными ⭐
Задайте вопрос на русском языке и получите ответ на основе **реальных данных** вашей системы:

```
"Какие организации имеют критический уровень риска?"
"Почему выросли нарушения по СИЗ в последние месяцы?"
"Что нужно сделать org_03 чтобы снизить аварийность?"
"Есть ли корреляция между нарушениями LOTO и авариями оборудования?"
```

## Локальный запуск

### Backend
```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env          # добавьте ANTHROPIC_API_KEY
uvicorn main:app --reload
```

```bash
cd frontend
```

## Алгоритмы

**Прогнозирование инцидентов**
Holt-Winters (ExponentialSmoothing) с трендом и аддитивной сезонностью — оптимален для нестационарных рядов с сезонными паттернами. Fallback на линейный тренд при недостатке данных. Доверительный интервал 80% (±1.28σ). Метрики на backtesting: R²=0.796, MAE=9.25. Горизонт: 3/6/12 месяцев.

**Risk Scoring**
Взвешенный индекс риска на организацию: 60% — тяжесть инцидентов (НС=10, Авария=7, ДТП=6...), 40% — количество нарушений по Коргау. Нормализация 0–100.

**Алерты (4 уровня)**
- 🔴 Критический: нарушений >2× среднемесячного за последние 30 дней
- 🟠 Высокий: один тип нарушения повторился >3 раз за 30 дней по одной организации
- 🟡 Средний: рост нарушений >15% к аналогичному периоду прошлого года
- 🟢 Низкий: снижение нарушений >20% (положительный сигнал)

**Экономический расчёт**
Прямые затраты на инцидент по типам (НС — 5 млн тг, Авария — 3 млн тг и т.д.) × ставка снижения после AI (38–50%) × коэффициент косвенных потерь 2.0 (косвенные = 200% от прямых, согласно ТЗ). Итого: ~197 млн тг/год на основе реальной статистики 820 инцидентов.

**AI Рекомендации и Чат**
Claude получает сводку последних 6 месяцев данных (топ типов инцидентов, причин, зон риска) и генерирует конкретные меры с приоритетами. Чат поддерживает историю диалога и имеет доступ к актуальной статистике системы.
